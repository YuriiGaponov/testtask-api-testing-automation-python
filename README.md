# testtask-api-testing-automation-python

## Описание проекта

Автоматизированная система тестирования API для проверки работы эндпоинта `/posts`. Проект реализует полный набор тестов для проверки CRUD-операций с использованием библиотеки `pytest` и `requests`.

## Установка и настройка

### Клонирование репозитория

```bash
git clone https://github.com/YuriiGaponov/testtask-api-testing-automation-python.git
cd testtask-api-testing-automation-python
```

### Создание виртуального окружения

1. **Создание окружения:**
```bash
python -m venv venv
```

2. **Активация окружения:**
* Для Windows:
```bash
venv\Scripts\activate
```
* Для Linux/MacOS:
```bash
source venv/bin/activate
```

### Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск тестов

Для запуска тестов выполните команду:

```bash
pytest tests/ -v
```

Флаг `-v` включает подробный вывод результатов выполнения тестов.

## Структура проекта

```
testtask-api-testing-automation-python/
├── tests/
│   ├── __init__.py          # Инициализация тестового пакета
│   ├── test_posts.py       # Основные тесты для эндпоинта /posts
│   └── conftest.py         # Фикстуры и общие настройки
├── requirements.txt        # Зависимости проекта
└── README.md               # Документация
```

## Основные компоненты

### conftest.py

Содержит:
* Базовую HTTP-сессию
* Настройки эндпоинта
* Тестовые данные
* Обязательные поля для проверки

### test_posts.py

Реализует тесты для:
* Получения списка постов
* Работы с отдельными постами по ID
* Создания новых постов
* Обновления постов (PUT и PATCH)
* Удаления постов

## Примеры тестов

### Успешные сценарии

* Получение списка постов (статус 200)
* Создание нового поста (статус 201)
* Обновление существующего поста (статус 200)
* Удаление поста (статус 204)

### Негативные сценарии

* Запрос несуществующего поста (статус 404)
* Некорректные данные при обновлении
* Удаление несуществующего поста

## Ожидаемый результат

При успешном выполнении все тесты должны завершиться со статусом **passed**. В выводе pytest будет отображена подробная информация о выполнении каждого теста, включая:
* Время выполнения
* Статус
* Сообщения об ошибках (при наличии)

## Рекомендации по использованию

* Перед запуском тестов убедитесь в корректности установленных зависимостей
* При необходимости модифицируйте тестовые данные в фикстурах
* Для просмотра подробного отчета используйте флаг `-v` при запуске pytest
* В случае ошибок проверяйте логи и сообщения assert-проверок

## Управление окружением

После завершения работы с проектом можно деактивировать виртуальное окружение командой:

```bash
deactivate
```
